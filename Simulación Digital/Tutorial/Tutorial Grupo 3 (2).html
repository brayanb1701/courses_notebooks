Qu es R?

R un lenguaje orientado a objetos dedicado al nalisis estadstico y grfico. Es un lenguaje interpretado y no compilado, lo cual significa que los comandos escritos en el teclado son ejecutados directamente sin necesidad de construir ejecutables. Todas las acciones en R se realizan con objetos que son guardados en la memoria activa del ordenador, sin usar archivos temporales.

R hace uso de libreras o paquetes, los cuales fortalecen el funcionamiento del lenguaje. Algunas funciones elementales para el manejo de estos son:

  • install.packages(): para instalar libreras
  • library(): cargar determinada librera
  • help(): obtener informacin acerca de una funcin de cierta librera

Estructuras de Control

R al igual que muchos lenguajes de programacin tambien posee unas estructuras que permiten controlar el flujo de ejecucin de una secuencia de comandos ya sea mediante la toma de una decisin o la ejecucin en un bucle definido o indefenido. Estas estructuras son conocidas como Estructuras de Control.

Las estructuras de control ms utilizadas en R son:
- if/else: nos permite tomar una decisin respecto a qu cdigo ejecutar dependiendo de que una condicin se cumpla o no.
- switch: toma la desicin dependiendo del valor de una variable. - for: Permite la ejecucin de un bucle un nmero de veces determinado.
- while: Ejecuta un bucle mientras se cumple una determinada condicin.
- repeat: Permite que un bucle se ejecute indefinidamente y solo se detiene mediante el uso del comando break.
- break: Este comando sirve la detener la ejecucin de un bucle.
- next: Nos permite saltarnos a la siguiente ejecucin de un codigo dentro de un bucle.

IF/ELSE

Nos permite actuar en funcin de una decisin. Existen tres formas de usar esta estructura de control. Miremos su sintaxis en cada una de estas formas:

  • Usar solo el if, esta forma se utiliza cuando solo queremos ejecutar un codigo si la condicin es verdadera y si llegase a ser falsa, se contina con la ejecucin normal sin ejecutar el codigo dentro del if.
...
if(<condicion>) {
  # Bloque de cdigo si la condicin es verdadera
}
...
  • Usar el if y el else, esta forma se utiliza cuando se necesita evaluar una condicin y dependiendo de que sea falsa o verdadera se ejecuta un bloque de cdigo u otro bloque de cdigo.
...
if(<condicion>) {
  # Bloque de cdigo si la condicin es verdadera
} else {
  # Bloque de cdigo si la condicin es falsa
}
...
  • Usar el if, el else y else if, aqu se van evaluar varias condiciones y dependiendo de cual sea falsa o verdadera se ejecuta un bloque de cdigo y si ninguna condicin es verdadera se ejecutar el bloque de cdigo del else. Esta estructura se usa poco debido al uso del SWITCH debido a que esta tiene un uso similar.
...
if(<condicin1>) {
  # Bloque de cdigo si la condicin1 es verdadera
} else if(<condicin2>) {
  # Bloque de cdigo si la condicin2 es verdadera
} else {
  # Bloque de cdigo si la condicin1 y condicin2 son falsa
}
...

Miremos un ejemplo:

x <- runif(1,0,10) ## creamos un numero aleatorio entre 0 y 10
print(x);
[1] 4.992102
if( x > 5 ){
  print("El numero es mayor de 5");
} else if(x == 5) {
  print("El numero es igual que 5");
} else {
  print("El numero es menor que 5")
}
[1] "El numero es menor que 5"

SWITCH

Tiene un uso similar al IF/ELSE con la diferencia de que SWITCH toma la decisin dependiendo del valor de una variable y ejecuta un bloque de cdigo en funcion de cada valor posible de dicha variable.

Su sintaxis es la siguiente:

...
switch(<EXPR>,
      <valor_1> {# cdigo}
      ....
      <valor_n> {# cdigo})
...

Ejemplo:

x <- 1:10;
type <- 'mean';
switch(type,
        mean = mean(x),
        median = median(x),
        sd= sd(x))
[1] 5.5

WHILE

El bucle while comprueba una condicin y si esta es verdadera ejecuta el bloque de cdigo. Una vez finalice la ejecucin del cdigo, vuelve a comprobar la condicin y si es verdadera procede nuevamente a la ejecucin del bloque de codigo. Finaliza cuando la condicin sea falsa o cuando encuentre un break.

Su sintaxis es la siguiente:

...
while(<condicin>) {
  # Bloque de cdigo
  ...
}
...

REPEAT

La estructura repeat ejecuta un bucle infinito, nicamente detenido por un break.

Su sintaxis es la siguiente:

...
repeat {
  # Bloque de cdigo
  ...
  if(<condicin de parada>) break;
}
...

Ejemplo:

cont <- 9;
repeat {
  print(cont);
  cont <- cont - 1;
  if(cont < 5) break;
}
[1] 9
[1] 8
[1] 7
[1] 6
[1] 5

NEXT / BREAK

El next se utiliza para finalizar un ciclo de un bucle en ejecucin y pasar al siguiente ciclo.

Ejemplo:

Imprimir todos los numeros pares de 1 a 10.

for(i in 1:10) {
  if(i %% 2 != 0) next; # Si el numero no es par, no ejecuta el print(i) y se salta al otro ciclo.
  print(i);
}

[1] 4
[1] 6
[1] 8
[1] 10

El break se utiliza cuando se quiere finalizar por completo un bucle sin que la condicin para terminarlo se cumpla.

cont <- 0
while(cont < 10) {
  x <- runif(1,0,10);
  print(cont);
  cont <- cont+1
  print(x)
  if(x < 5) break;
}

[1] 1

Importar Datos

Debido a la importancia del manejo de datos, R permite importar conjuntos de datos que tengan algn tipo de organizacin, comunmente conocido como tablas ya que se organizan por medio de filas y renglones. Para esto, el nico requisito es precisamente que el archivo este organizado por renglones y que, para identificar las columnas, cuente con un separador (Ej: , ; | ).

R cuenta con funciones del paquete base que permiten realizar esta labor, que son:

  • read.table(file, header = FALSE, sep = ", quote =", dec =.", )
  • read.csv(file, header = TRUE, sep = ,, quote = ", dec = ., fill = TRUE, comment.char = "", )
  • read.csv2(file, header = TRUE, sep = ;, quote = ", dec = ,, fill = TRUE, comment.char = "", )
  • read.delim(file, header = TRUE, sep = , quote =", dec =., fill = TRUE, comment.char =", )
  • read.delim2(file, header = TRUE, sep = , quote =", dec =,, fill = TRUE, comment.char =", )

Como se puede observar, todas son muy similares entre s, cambiando los valores predeterminados de unos cuantos parmetros lo cual las hace muy flexibles. Por otro lado, se encuentra la librera readr>, la cual contiene 4 funciones equivalentes a las mencionadas anteriormente, que son:

  • read_csv(): para leer archivos con coma (,) como separador.
  • read_csv2(): para leer archivos con punto y coma (;) como separador.
  • read_tsv(): para leer archivos con tabulador () como separador.
  • read_delim(,sep =|): para leer archivos con cualquier tipo de separador.

Las funciones de la librera presentan unas cuantas ventajas respectos a las de base:

  • Generalmente son mucho ms rpidas (~10x) que sus equivalentes. Los trabajos que tienen un tiempo de ejecucin prolongado poseen una barra de progreso para que puedas ver qu est ocurriendo. Si solo te interesa la velocidad, prueba data.table::fread(). No se ajusta tan bien con el tidyverse, pero puede ser bastante ms rpido.

  • Producen tibbles, no convierten los vectores de caracteres a factores, no usan nombres de filas ni distorsionan los nombres de columnas. Estas son fuentes comunes de frustracin al utilizar las funciones de R base.

  • Son ms reproducibles. Las funciones de R base heredan ciertos comportamientos de tu sistema operativo y de las variables del ambiente, de modo que importar cdigo que funciona bien en tu computadora puede no funcionar en la de otros.
